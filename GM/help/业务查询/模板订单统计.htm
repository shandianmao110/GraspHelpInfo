模板订单统计

   

     模板订单统计是针对由生产模板组装而来的商品的统计，例如“联想电脑”由4样配件组成，当销售订单有“联想电脑”100台时，在模板订单统计中可以查询到只针对“联想电脑”的4样配件的库存状况及所需数量， 并且可根据所需配件数量生成进货单或进货订单草稿， 如图所示 。

 

     图中，选中模板名称为“电脑组装”的模板，双击组装商品“联想电脑”，在下部分表体中显示出相关配套商品的信息。“配套数量”、“配套商品”是由生产模板取来的。

　

表体内容说明：

     配套数量 ：取自模板订单中组装商品所需的配件数；

     账面库存数量：当前库存中的商品数量；

     进货订货数量：指进货订单中未到货数量；

     销售订货数量：指销售订单中未到货数量；

     供应商： 取自基本信息中商品所对应的默认供应商；

     计算数量= 账面库存数量 C 销售订货中未到货数量；

        当计算数量< 0， 说明库存数量不够，需要对配件进行加工。配套商品表体中显示“模板所需数量”一列；

        当计算数量>= 0， 说明库存数量充足，不需要对配件进行加工。配套商品表体中不显示“模板所需数量”一列；

     可用数量 = 账面库存数量 + 进货订货数量 C 销售订货数量  （进货订货数量、销售订货数量均为未到货数量），可点击“公式”按钮进行修改，如图所示 。

    

     模板所需数量 = 计算数量 x   配套数量

     剩余数量 = 可用数量 C 模板所需数量

     确认数量：用户填写的需要进货的商品数量；

　

按钮说明：

     修改：修改商品对应的供应商，此处的修改仅是在生成草稿时生效，不会修改基本信息中默认的供应商。

     生成草稿： 根据用户输入的确认数量，生成商品的进货单或进货订单草稿。对应相同供应商的商品将会生成一张草稿。

     公式：可以修改“可用数量”计算的公式。
