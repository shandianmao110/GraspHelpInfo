SQL2005安装过程

　

数据库版本：请使用SQL2005标准版（Standard））

安装过程中有可能出现的问题：

A、在我们这安装的时候发现了这样一个问题，在安装过程中报了一个ID：1706的错，导致安装失败，并安装的时候报 SQL  Native  Client错误，经查，这个是操作系统中的office软件问题。

解决办法步骤如下：

1、  重新安装或修复安装office

2、  删除已安装的SQL 2005组件（SQL Native Client必须删除）

3、  重新安装SQL 2005

 

B、在安装的时候有可能出现性能监视器计数器要求检查不能通过的情况，解决办法如下：

1、  在开始菜单－运行栏输入regedit，打开注册表编辑器

2、  找到路径HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows NT \ CurrentVersion \ Perflib

3、  在Perflib下面会有004和009两个目录，分别对应中文和英文操作系统，如果是中文操作系统，请使用004目录，如果是英文操作系统，请使用009目录

4、  将对应目录（004或009）下的Counter打开(双击)，如图所示，把红圈所示数字拷贝下来，注意：是编辑框中的最后一个数字

            

　

5、 打开Perflib目录下的Last Counter项，在数值数据中填入刚拷贝的数据，然后选择十进制

6、 参照第四和第五步骤，将对应目录（004或009）下的Help项的值拷贝修改Perflib目录下的Last Help项的值即可

 

C、在安装过程中，有可能会报磁盘空间不足的问题，但用户查看所安装的磁盘却还有很大的空间，这是什么原因呢？这个是由于你的系统盘空间不足所引起的，虽然用户可能没安装在系统盘，但是在安装的时候系统盘会存放很多临时文件，实际上安装的时候需要很大的系统盘空间，在安装的时候有一个办法可以查看安装所需的磁盘空间，避免因磁盘空间不足而无法安装。

如下图所示，当安装过程进行到图0－1所示部分时，在更改安装路径之后点击红圈所示磁盘开销进入图0－2，在图0－2中保证差额列所有数字大于0即可正常安装。
